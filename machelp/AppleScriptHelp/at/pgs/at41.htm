引用 Finder 中的项目

    要控制 Finder 项目，如文件和文件夹，需要在命令语言中指定该项目。 有多种方法可以引用项目。 另外，您可以使用 AppleScript 术语来指定多组文件。

引用类型

    在命令语言中有三种方法可以引用文件或文件夹。 每种类型的引用均需指明项目的名称、项目的类型和在计算机中的位置。

    Finder 引用：
    Finder 引用以文件或文件夹的名称开始，然后给出项目的位置（即包含该项目的“一连串文件夹”），并以磁盘结束。 下面是一个例子：

    file "Art.gif" of folder "Images" of disk "Macintosh HD"

    Finder 引用可以包含以下特殊文件夹和磁盘的名称：
    • system folder（系统文件夹）
    • fonts folder（字体文件夹）
    • desktop（桌面）
    • extensions folder（功能扩展文件夹）
    • preferences folder（预置文件夹）
    • apple menu items folder（苹果菜单项目文件夹）
    • startup disk（启动磁盘）

    您不需要在这些迅捷中使用引号。 下面是两个例子：

    file "Art.gif" of folder "Images" of startup disk

    file "My Prefs" of preferences folder

    路径引用：
    路径引用以磁盘的名称开始，给出文件夹的名称，以文件或文件夹的名称结束。 路径中的每一个项目以冒号分开。 文件夹的路径引用以冒号结束。 例如：

    file "Macintosh HD:Images:Art.gif"

    folder "Macintosh HD:Images:"

    替身引用：
    替身引用与路径引用的形式一样，但它是以“alias” 开头（而不是以“file”或“folder”开头）：

    alias "Macintosh HD:Images:Art.gif"

    alias "Macintosh HD:Images:"

    命令语言中的替身引用与 Finder 中替身的工作方式大致相同。 Finder 和路径引用是绝对的，所以如果移动了命令语言所引用的项目，则不能执行该命令语言。 如果使用替身引用，由于替身包含文件的位置，因此即使文件被移动，命令语言仍可以正常工作。

    如果引用远程磁盘上的项目：
    AppleScript 在对命令语言进行编译时，会尝试找到远端磁盘上的替身引用。 远端磁盘必须可用，而且您必须提供访问该磁盘所需的相应的用户名和密码。

贴引用

    除了键入引用外，您也可以拷贝和贴引用。 要将对文件或文件夹的 Finder 引用贴进命令语言：
    1 在 Finder 中选择一个文件或文件夹，然后打开“编辑”菜单并选择“拷贝”。
    2 在“命令语言编辑程序”中，将插入点放在命令语言中需要引用出现的地方。
    3 打开“编辑”菜单并选取“贴引用”。

标识项目

    AppleScript 包含的术语可让您以不同方法指定多组项目。

    Every:
    使用 Every 来引用一个位置的所有文件或文件夹。 例如：

    set my_list to every file of folder "Process Files"

    Whose 和 Where Its:
    使用 Whose 和 Where Its 根据一个条件引用文件或文件夹。 例如：

    set my_list to every file of folder "Process Files" whose¬
    name contains "project"

    set my_list to every file of folder "Process Files" where its¬
    name contains "project"

    这两个语句会获得相同的结果。

    此例子根据标签属性引用文件夹：

    set my_list to every file of the startup disk whose¬
    label index is not 0

    And 和 Or:
    使用 And 和 Or 可根据几个条件来引用项目。 例如：

    set my_list to every file of folder "Process Files" whose¬
    name contains "project" and label index is not 0

    Entire Contents:
    在 Containers and Folders（容器和文件夹）术语套中，Finder 定义了 Container 对象，它包含 Entire Contents（全部内容）的属性。 因为文件夹和磁盘是容器，您可以使用该属性来引用文件夹中的所有项目。 例如：

    set my_list to every file of the entire contents of folder¬
    "Process Files" of startup disk whose name label index is 0
