使用“個人網頁共享”的問題

    如果無法啟動“個人網頁共享”：
    • 您可能需要調整 TCP/IP 設定。請確定您的 IP 位址格式正確。如果需要 IP 位址、網域名稱或其他網路資訊，請詢問您的網路管理者。

    如果其他人無法連接您的伺服器：
    • 請確定“網頁共享”控制面板的“個人網頁共享”已打開。
    • 如果您的伺服器已設定使用檔案共享權限來控制取用（已選取“網頁共享”控制面板中“用“檔案共享”來控制使用者取用”），請確定“檔案共享”控制面板中的檔案共享已打開。
    請替我打開“檔案共享”控制面板。
    • 如果您的伺服器已設定使用檔案共享權限，請確定“網頁”檔案夾已打開。
    如果“網頁”檔案夾目前尚未共享，且您正在使用檔案共享將取用權限設定至該檔案夾，一直要到您打開檔案共享後，其他人（包括您個人在內）才能取用“網頁”檔案夾。
    • 請確定“網頁”檔案夾置於硬碟根目錄下五層子目錄之內。
    • 確定 TCP/IP 連線設定無誤。如果需要協助，請諮詢網路管理者。
    • 訪客使用的“網頁”檔案夾的位址可能錯誤。請確定訪客知道正確的網域名稱或 IP 位址。您可以使用“網頁共享”控制面板的“編輯”清單中“拷貝我的位址”指令，將完整的位址發送給其他使用者。
    • 您的“網頁”檔案夾可能使用非 80（預設值）的埠編號。您可以打開“網頁共享”控制面板，選取“編輯”清單中的“預置…”，然後按“選項”標籤一下，來檢查埠編號。（如果已更改為異於 80 的埠編號，新的編號即會出現於 IP 位址的結尾。）
    • 請確定“網頁”檔案夾具有唯一指定的 IP 位址。勿使用非標準“網頁”檔案夾的 IP 位址。有些 IP 位址（例如：“0.0.0.0”或“1.1.1.1”）是保留給網路硬體（如：路由器和閘道器）所使用。如果不瞭解該如何指定“網頁”檔案夾的 IP 位址，請聯絡網路管理者。
