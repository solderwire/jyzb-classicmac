关于使用“个人 Web 共享”的问题

    如果不能打开“个人 Web 共享”：
    • 您可能需要调整 TCP/IP 设置。请确保您的 IP 地址格式是否正确。如果需要 IP 地址、域名称或其他网络信息，请向网络管理员咨询。

    如果其他人不能连接到您的服务器：
    • 请确保“Web 共享”控制板中的“个人 Web 共享”已被打开。
    • 如果服务器被设置为使用文件共享权限来控制访问（在“Web 共享”控制板中选定了“使用‘文件共享’来控制用户访问”），请确保“文件共享”控制板中的文件共享已打开。
    请为我打开“文件共享”控制板。
    • 如果服务器被设置为使用文件共享权限，请确保 Web 文件夹已设为共享。
    如果您的 Web 文件夹当前没有设为共享，而您又使用文件共享给它设置访问权限，在您打开文件共享前，没有人（包括您自己）能访问这个 Web 文件夹。
    • 请确保您的 Web 文件夹位于硬盘根目录下不超过五级的子目录中。
    • 确保您已正确设置 TCP/IP 连接。如果需要帮助，可以向您的网络管理员咨询。
    • 访客可能没有使用您 Web 文件夹的正确地址。请确保他们有正确的网域名称或IP 地址。您可以通过使用在“Web 共享”控制板“编辑”菜单中的“拷贝我的地址”命令向其他用户发送您的完整地址。
    • 您的 Web 文件夹也许使用了 80（预设值）以外的端口号。您可以从“编辑”菜单选择“预置…”并按“选项”标签，以检查“Web 共享”控制板中的端口号。（如果端口编号不是 80 ，新的序号会出现 IP 地址的末尾。）
    • 请确保您的 Web 文件夹有一个唯一指定的 IP 地址。不要让您的 Web 文件夹使用非标准的 IP 地址。有些 IP 地址（例如“0.0.0.0”或“1.1.1.1,”）是为网络硬件保留的，例如路由器和网关等。如果您不清楚如何给 Web 文件夹指定 IP 地址，请与您的网络管理员联系。
