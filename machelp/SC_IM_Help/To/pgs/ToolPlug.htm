Input Method Plug-in
(输入方法插件)

    Input Method Plug-in 是 Mac OS 简体输入方法内一项新加入的功能，用户可以容易地利用这项功能制作自己的输入方法。

    由于这个 Plug-in 功能可以使用文本格式的文件，故此用户可以轻易地将现有 PC 上的输入方法转放在 Mac OS 上使用。另一方面，用户也可以自己定义输入码，然后利用建立文字文件的方法，制作自己适用的输入方法

制作 Plug-in：

    1 用户首先制作一个纯文本文件（请参阅下节的格式举例）。您可以使用任何文字处理程序，但必须把文件存储为纯文本、即“TEXT”类型。
    2 把纯文本文件拖移到 Input Method Plug-in Converter 的图象上。
    Input Method Plug-in Converter 在 Apple Extras 的 Chinese Utilities 内的 Input Method Plug-in 文件夹内。
    Input Method Plug-in Converter 程序会把纯文本文件转换成为一个简体输入方法可以使用的文件，存储在原来纯文本文件同一个文件夹内。文件名称是原文件名称加上“.dat”。
    3 把已转换的文件（即名称后面是“.dat”的文件）拖移到“系统文件夹”内的“功能扩展”内的 Input Method Plug-in 文件夹内。
    4 重新启动。

所制作的 Plug-in 名称会出现在输入方法菜单和调板内。（如图例中最下方的“西北变调拼音”。）

文本文件格式

    为使系统能正确地使用您所制作的 Plug-in，请遵照指定的文件格式；您可使用系统提供的版样。文件格式主要分为两部分：标题（header）和内文。

    标题

    标题的每行定义为：

    METHOD：现以 TABLE 用作说明

    ENCODE： 输入方法的语系，即 GB 或 BIG-5

    PROMPT： Plug-in 的名称，最多可使用 32 个字符。Plug-in 名称在重新启动后会出现在“输入法”菜单的最下面。

    VERSION： Plug-in 的版本，最多可使用 8 个字符。

    ACTIVATEKEY：定义激活键。

    MAXCODE：最多的输入键数。这项应为数字。

    VALIDINPUTKEY： 在这行定义有效的输入键。应少于 128 个字符。

    TERMINPUTKEY： 在此定义终止输入键。当按下这些键时，输入方法会强行将键转换为中文。

    BEGINCHARACTER： 开始文件内文，是字符和词组定义部分。

    ENDCHARACTER：结束字符和词组定义部分，应与 BEGINCHARACTER 一起使用。

    文件内文

    文件内文紧接著标题，以 BEGINCHARACTER 行开始，以 ENDCHARACTER 行结束。

    从 BEGINCHARACTER 行开始至 ENDCHARACTER 行的文件内文内，用户可定义代码行。代码行定义为：

    codeline ::= keystring, blanks,{character/phrase,(,/，)}* linebreak

    {...}* 内的内容会出现一次或超过一次；而文字／词组／句子应以罗马语系（单字节）的逗号分隔开。一个句子最多可使用一百个中文字。

    以“#”字符开始的任何行都是注释，Plug-in Converter 转换程序会略过。

    文字文件样板

    以下以文中所提到的“西北变调拼音”为例 ：

    # begin file header
    METHOD: TABLE
    ENCODE: GB
    PROMPT: 西北变调拼音
    VERSION: 1.0
    ACTIVATEKEY: 2
    MAXCODE: 7
    #
    # input key definitions
    VALIDINPUTKEY: 0123456789;abcdefghijklmnopqrstuvwxyz
    TERMINPUTKEY: 12345
    # the following line must not be removed
    BEGINCHARACTER
    # any blank lines is ignored
    # each line can contain single characters or phrases
    # the delimiter could be either a one-byte or a two-byte comma
    a1 邓,呵,阿,腓
    a2 苡
    a3 阿
    a4 阿,腓
    a5 啊,阿
    ai1 哀,埃,挨,哎,□,□,唉
    # add more code lines below
    ENDCHARACTER
